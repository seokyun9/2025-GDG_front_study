## 서버? 클라이언트?

#### 서버
: 서빙하는 사람... 고객이 오면 서빙하는 역할!   
클라이언트가 request(요청)를 보내면 Response(응답)를 보낸다.
서버<->회사(정보) 에서 동작하는 부분을 다루는 개발영역을 **백엔드** 라고 한다.
#### 클라이언트
: 서빙을 받는 고객!
서버에게 데이터를 요청 (request)
유저<->클라이언트 즉, 사용자가 직접 상호작용하는 웹 페이지의 화면을 다루는 개발 영역을 **프론트엔드** 라고 한다.

#### API 
클라이언트-서버 간 데이터 주고받는 통로의 역할    
서로 다른 소프트웨어인 두 역할 간 상호작용을 돕는 인터페이스

### fetch 함수

## useEffect
setState로 데이터를 업데이트하려하면 무한루프에 빠진다!
( 컴포넌트 코드 시작 -> API fetch-> 데이터 state에 대입 -> state변경? -> 업데이트! -> 컴포넌트 코드 재시작... )

=> 외부 영향 처리를 위해 useEffect Hook이 필요!

    외부 시스템과 컴포넌트를 동기화하는 React Hook
    컴포넌트가 렌더링 될 때 특정 작업을 실행할 수 있게 한다.
